概念

  • 浅拷贝：仅仅拷贝基本类型的值，如果不是基本类型则只拷贝其对象的引用。
  • 深拷贝：拷贝生成的对象和原来对象中的值完全相同，如果拷贝时遇到非基本类型的情况，会给非基本类型的对象重新分配内存空间，将对象的引用指向新分配的空间。
  • Java中的clone()方法
    • 对任何的对象x，都有x.clone() !=x//克隆对象与原对象不是同一个对象
    • 对任何的对象x，都有x.clone().getClass()= =x.getClass()//克隆对象与原对象的类型一样
    • 如果对象x的equals()方法定义恰当(equals方法比较的是值而不是地址的情况)，那么x.clone().equals(x)应该成立。

注意:String对象是无法实现深拷贝的。

使用序列化来实现深拷贝

为了避免重写复杂对象的clone()方法，我们可以通过序列化的方式来实现深拷贝。

序列化的过程是把对象写到流里的过程，反序列化是把对象从流中读出来的过程。

写在流里的是对象的一个拷贝，原对象仍然存在于JVM中。

在Java语言里深拷贝一个对象的过程：先使对象实现Serializable接口，然后把对象(实际上只是对象的一个拷贝)写到一个流里，再从流里读出来，便可以重建对象。

这样做的前提是对象以及对象内部所有引用到的对象都是可序列化的，否则，就需要仔细考虑那些不可序列化的对象或属性能否设置成transient，从而将之排除在复制过程之外。

注意Cloneable与Serializable接口都是marker Interface，也就是说它们只是标识接口，没有定义任何方法。
